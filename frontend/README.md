## Project Scripts

Within the project folder, the following npm commands are available:

Starts the application in **development mode**.

* The app runs locally at **[http://localhost:3000](http://localhost:3000)**

* Warnings and linting messages will be shown in the console

---

### ðŸ“¦ `npm run build`

Generates an **optimized production build**.

* Output files are placed inside the `build/` directory
* Code is minified and optimized for performance
* Asset filenames include hashes to improve caching

After completion, the project is ready for deployment.

---

### âš  `npm run eject`

> **Important:** This is a permanent action and cannot be undone.

* Removes the default abstraction layer
* Exposes all configuration files (Webpack, Babel, ESLint, etc.)
* Allows complete customization of the build process

This step is optional and generally unnecessary for small to medium-scale applications.
